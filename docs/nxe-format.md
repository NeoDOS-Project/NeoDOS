# NXE Format Reference

> **Container:** ELF64 (ET_DYN / PIE)
> **Extension:** `.NXE` (on disk: `.nxe`)
> **Kernel loader:** `neodos-kernel/src/elf.rs`

---

## 1. Base Format

NXE binaries are standard ELF64 `ET_DYN` (Position-Independent Executable) for x86_64, compiled with `relocation-model=pie`, linked at address `0`, and loaded with ASLR into the user slot window `0x400000..0x2400000`.

| Aspect | Detail |
|--------|--------|
| File format | ELF64 |
| Magic | `\x7fELF` |
| Type | `ET_DYN` (3) |
| Machine | `EM_X86_64` (62) |
| Endianness | Little-endian |
| Entry point | `_start` (no arguments, no return) |
| Link address | `0` (PIE, resolved at load time) |
| Relocation model | `pie` |
| Max binary size | 64 KB (65536 bytes) |
| Load address | Random slot in `[0x400000, 0x2400000)` (ASLR) |
| Slot size | 128 KB per process (64 KB code + 64 KB stack) |
| Stack size | 64 KB, grows downward from slot top |
| Heap | 2 MB per process, at `0x10000000` |
| Linker script | `libneodos/user.ld` |
| Build target | `x86_64-unknown-none` |

## 2. NeoDOS Metadata Note

The `.note.neodos` ELF note section (type `PT_NOTE` / `SHT_NOTE`) carries metadata about the NXE binary. The kernel ELF loader ignores it. Tools read it for display and validation.

### Note header (ELF note format)

| Offset | Size | Field | Value |
|--------|------|-------|-------|
| 0 | 4 | `namesz` | 8 (`"NeoDOS\0\0"`) |
| 4 | 4 | `descsz` | N (metadata block size) |
| 8 | 4 | `type` | 1 (`NT_NEODOS_META`) |
| 12 | 8 | `name` | `"NeoDOS\0\0"` |
| 20 | N | `desc` | TLV entries |

### TLV entry format

| Offset | Size | Field |
|--------|------|-------|
| 0 | 2 | `tag` (u16 LE) |
| 2 | 2 | `length` (u16 LE) |
| 4 | L | `value` |

### Defined tags

| Tag | ID | Type | Description |
|-----|-----|------|-------------|
| `NEM_PRODUCT_NAME` | 0x0001 | String | Human-readable product name |
| `NEM_PRODUCT_VER` | 0x0002 | String | Product version (semver) |
| `NEM_FILE_VERSION` | 0x0003 | String | File version (semver) |
| `NEM_DESCRIPTION` | 0x0004 | String | Short description |
| `NEM_AUTHOR` | 0x0005 | String | Author/maintainer |
| `NEM_LICENSE` | 0x0006 | String | SPDX license identifier |
| `NEM_ARCH` | 0x0007 | String | Target architecture |
| `NEM_SUBSYSTEM` | 0x0008 | u32 LE | 0=NATIVE, 1=CONSOLE, 2=GUI |
| `NEM_LANGUAGE` | 0x0009 | String | Default locale tag |
| `NEM_MIN_KERNEL` | 0x000A | String | Minimum kernel version |
| `NEM_DEPENDENCIES` | 0x000B | String | Semicolon-separated NXL deps |
| `NEM_RES_DIR` | 0x000C | String | Resource directory in package |
| `NEM_MANIFEST_FLAGS` | 0x000D | u32 LE | Flags bitmask |
| `NEM_ORIGINAL_FILENAME` | 0x000E | String | Original .NXE filename |
| `NEM_BUILD_TIMESTAMP` | 0x000F | u64 LE | Unix build timestamp |
| `NEM_GIT_HASH` | 0x0010 | String | Git commit hash |

### Manifest flags

| Bit | Name | Description |
|-----|------|-------------|
| 0 | `CORE` | Critical system component |
| 1 | `BOOT` | Loaded at boot time |
| 2 | `NOASLR` | Disable ASLR |
| 3 | `SINGLETON` | Only one instance allowed |

## 3. Tool Inspection

```bash
# Display brief info
nxinfo ver.nxe

# Display full metadata
nxinfo ver.nxe --metadata

# Display all ELF headers
nxinfo ver.nxe --headers

# Display sections
nxinfo ver.nxe --sections

# JSON output
nxinfo ver.nxe --json

# Validate NXE format
nxinfo ver.nxe --check

# Technical dump
nxdump ver.nxe --segments
nxdump ver.nxe --relocs
```

## 4. Adding Metadata to an NXE

Metadata is embedded via a `#[link_section = ".note.neodos"]` static in the Rust source. A build helper generates the binary TLV blob.

```rust
// Generated by build.rs or manual
#[used]
#[link_section = ".note.neodos"]
static NXE_META: [u8; 64] = *b"\x08\x00\x00\x00...NeoDOS\0\0...";
```

For host-side injection (without modifying source), `neodev` can use:
```bash
objcopy --add-section .note.neodos=metadata.bin input.nxe output.nxe
```
